CWTS Inventory System API

A REST API to manage inventory items used for CWTS activities.

How to Access

Copy these URLs to access the endpoints:

  • https://cwtsinventory.onrender.com/items
  • https://cwtsinventory.onrender.com/logs

Endpoints

Items

  • /items – GET all inventory items

Logs

  • /logs – GET all inventory logs

Example Item Response

{
        "id": 1,
        "name": "Broom",
        "category": "Cleaning Supplies",
        "quantity": 10
    }

Example Log Response

{
        "date": "2025-07-03",
        "log_id": 1,
        "name": "Dust Pan (Pangdakot)",
        "qty": 5,
        "type": "OUT"
    }

Performing CRUD Operations Using Postman

You can test and interact with this API using Postman. Below are the available operations:

Available Operations for /items:

  • POST /items – Add a new item
  • GET /items – Retrieve all items
  • PUT /items/<item_id> – Fully update an item’s quantity
  • PATCH /items/<item_id> – Partially update item name or quantity
  • DELETE /items/<item_id> – Delete an item by ID

Available Operations for /logs:

  • POST /logs – Add a new log and update item quantity
  • GET /logs – Retrieve all logs with item names
  • PUT /logs/<log_id> – Fully update a log entry and adjust inventory
  • PATCH /logs/<log_id> – Partially update a log (type, qty, date)
  • DELETE /logs/<log_id> – Delete a log and revert its effect on inventory

Make sure to use raw JSON format in the Body tab when testing POST, PUT, or PATCH requests in Postman.
